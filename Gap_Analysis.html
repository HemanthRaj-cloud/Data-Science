The Real Gaps — Brutally Honest

These are the specific things that will get you rejected if not addressed

Hard Truth
None of these gaps mean you're unqualified. They mean you're not yet interview-ready. Every single one of these can be closed in 4–10 weeks with daily practice. The candidates who get offers aren't smarter — they're just more deliberate about closing these specific gaps.
🚨
SQL — You Know Basics, But Will Fail the Interview Round
You can write SELECT queries and basic JOINs — that's it. FAANG DS SQL rounds test window functions, CTEs, self-joins, correlated subqueries, LEAD/LAG, CASE WHEN chains, and multi-table analytical queries. These aren't edge cases — they are the standard. Without these, you will fail the SQL round even if you ace everything else. This is the #1 gap to close first because it's the most predictable and testable blocker.
🔴 Urgent Start Day 1 4–5 weeks to interview-ready
💀 Interview Killer: "Write a query to find users who logged in 3 consecutive days" — this requires window functions. Without them, you can't answer it.
🚨
Coding / LeetCode — You've Never Practiced Algorithmic Thinking
Your Python is functional — you can write scripts and work with Pandas — but DS coding interviews test algorithmic problem-solving under time pressure: hash maps, two pointers, sliding window, recursion, binary search. These patterns don't come from data science coursework. You need to solve ~120–150 problems before your first interview. This takes deliberate, consistent daily practice and cannot be rushed.
🔴 Urgent Start Day 1 6–8 weeks to reach interview-ready
💀 Interview Killer: "Given an array, find the two numbers that sum to a target" — sounds easy. Under pressure, without hash map pattern practice, many candidates freeze.
⚠️
ML Explanation — You Know It, But Can't Explain It Interview-Style
You've studied Random Forest, XGBoost, regression, etc. in your M.S. — but there's a big difference between understanding a concept and explaining it clearly, precisely, and confidently under interview pressure. "Explain gradient boosting in simple terms." "Why does L1 regularization produce sparse weights?" "How would you handle class imbalance in production?" — these require rehearsed, clear verbal articulation, not just conceptual understanding. You have the knowledge. You need to build the delivery.
🟡 High Week 2 onward 3–4 weeks of daily practice
✅ Fix: Record yourself answering 1 ML theory question per day. Play it back. Iterate. After 30 days you'll sound like an expert.

A/B Testing & Experimentation Design — A FAANG Core Skill You Lack
This is one of the most tested topics in DS interviews at Google, Meta, and Amazon — and it's almost entirely missing from your profile. Designing an A/B test from scratch, calculating sample sizes, interpreting p-values in a product context, identifying pitfalls (novelty effect, network effects, multiple testing), understanding when NOT to ship a result — these are day-1 skills for a DS at any FAANG company. Your statistics foundation from your M.S. will help here, but you need applied product framing on top of it.

Exactly What to Learn — In This Order

Prioritized by interview impact. Do not skip ahead.

Do Not Multitask
The biggest mistake candidates make is trying to learn everything at once. Go deep on one skill at a time. Finish Level 1 before starting Level 2. Shallow coverage of 10 skills loses to deep mastery of 5 skills every time in technical interviews.
1
Advanced SQL — Window Functions, CTEs, Analytical Queries
Start Today
This is the highest-probability interview killer on your list. 90% of DS interviews have a SQL round. Right now you'd fail it. The good news: SQL is the most learnable skill here — 4 weeks of daily practice transforms you from "knows basics" to "passes every SQL round."
Window Functions ROW_NUMBER / RANK / DENSE_RANK LAG / LEAD CTEs (WITH clause) Correlated subqueries Self-joins CASE WHEN (complex) Cohort / retention queries
Exact action: Open StrataScratch.com → filter by company (Meta, Google, Amazon) → solve 5 problems per day minimum. Start with Medium difficulty. Do NOT use hints for the first 20 minutes. After 150 problems you'll pass any DS SQL round. Use Mode Analytics SQL tutorial for window functions — it's the clearest free resource.
⏱ Time to interview-ready: 4 weeks at 5 problems/day
2
LeetCode — Arrays, Hash Maps, Two Pointers, Sliding Window (Python)
Start Day 1 Parallel
DS coding interviews are easier than SWE but still require ~5 core patterns. You don't need to know DP or graphs deeply — you need Arrays, Hash Maps, Two Pointers, Sliding Window, and Basic Recursion cold. The secret is pattern recognition, not raw intelligence. Once you've solved 20 hash map problems, you see the pattern everywhere.
Arrays & Strings Hash Maps (most common) Two Pointers Sliding Window Binary Search Basic Recursion Sorting algorithms Stack / Queue
Exact action: Go to LeetCode → sort by Easy → solve 5/day for 2 weeks. Then move to Medium, 3–5/day. When you get stuck, spend 20 min max then look at the solution — understand the pattern, don't just memorize it. Track your progress in a Notion doc. At 120 problems you're ready. Focus on Python exclusively.
⏱ Time to interview-ready: 6 weeks at 5 problems/day
3
ML Concepts — Verbal Explanation Practice (Not More Theory)
Week 2
You already know the theory. What you lack is the ability to explain it clearly, precisely, and conversationally under pressure. "Walk me through how a Random Forest works" is a fundamentally different challenge than understanding it from a textbook. This is a communication skill, not a knowledge skill. Build it through daily verbal practice.
Bias-Variance tradeoff Gradient Boosting intuition L1 vs L2 Regularization Overfitting solutions Class imbalance handling AUC-ROC interpretation Train/val/test split rationale Feature importance methods
Exact action: Pick 1 ML concept per day. Set a 5-minute timer. Explain it out loud like you're talking to a PM who is smart but not technical. Record on your phone. Play it back. Does it make sense? Is it under 3 minutes? Use "Ace the Data Science Interview" by Nick Singh — the best book for this exact type of practice.
⏱ Time to interview-ready: 3–4 weeks of daily 15-min practice
4
Scikit-learn End-to-End — Build Full ML Pipelines From Memory
Week 2
Stop reading about ML. Start building complete models — preprocessing → feature engineering → model selection → cross-validation → hyperparameter tuning → evaluation → interpretation — all in one clean notebook, from memory, in under 30 minutes. Do this on 3–4 different real datasets and you'll nail any take-home challenge or live coding DS round.
Pipeline + ColumnTransformer StandardScaler / OneHotEncoder CrossValidation (StratifiedKFold) GridSearchCV / RandomizedSearchCV SHAP values Confusion Matrix / ROC Curve Imbalanced-learn (SMOTE) Optuna (modern HP tuning)
Exact action: Pick the Titanic, Heart Disease, and Churn prediction datasets (all free on Kaggle). Build a complete sklearn Pipeline for each — from raw CSV to final trained model with evaluation report. Do it from a blank notebook, no copy-pasting. After 3 complete builds, this becomes muscle memory.
⏱ Time to interview-ready: 3 weeks building 1 full pipeline every 2 days
5
A/B Testing & Experimentation — Product Context Statistics
Week 3
Your M.S. stats knowledge is the foundation — now you need the product application layer. How to frame a business experiment as a statistical test. How to calculate sample size before running it. How to check for novelty effects, network effects, and p-hacking. How to report results to a PM clearly. This is one of the most reliably tested skills at Meta, Google, Amazon, and Airbnb.
Sample size calculation Power analysis (β, α) Multiple testing correction Guardrail metrics Network effects handling Novelty effect identification Bayesian A/B testing Confidence interval interpretation
